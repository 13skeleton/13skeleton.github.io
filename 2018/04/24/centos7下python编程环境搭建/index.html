centos7下python编程环境搭建

python3.6的安装

根据Python版本的不同，目前Python主要划分为两大类，即：Python2.x与Python3.x。其中，Python2.x在各种项目中应用广泛，得到众多社区的支持，而Python3.x作为最新的版本，逐渐成为未来Python语言的发展趋势，开始被应用于各种项目。可见，为满足开发项目对Python版本的要求，需要在当前操作系统环境中配置独立的运行环境，基于Python的virtualenv工具能够满足该任务要求。另外，通过配置独立Python环境可以避免直接安装另一Python版本造成系统内某些脚本无法正常运行的问题。

centos7默认安装的时python2.7，我在windows下采用的集成环境是anaconda3，当时的发行版是python3.6。因此，我们首先要在服务器中安装python3.

第二步，python3.6的编译安装

编译安装完成后，可以在制定目录/usr/local/bin中找到python3.4的语言版本，并且通过执行"python --version"后比较，可发现系统Python默认解释器版本并未改变。

注意要点:

1）“./configure”一行命令用于生成Makefile文件，用于下一阶段的编译步骤，其中，“--prefix”选项用于指定软件安装目录，以上述脚本为例，已知指定软件安装目录为“/usr/local”，相关执行文件则安装在其下的bin目录中，即：/usr/local/bin，资源文件则安装在其下的share目录中，即：/usr/local/share；“--with-ensurepip”选项用于确保pip工具能够正确安装，为推荐选项。

2）“make install”一行命令用于安装已经编译完成的软件代码，一般要求以root身份执行。

第三步，virtualenv的安装和使用

Python的虚拟环境可以使一个Python程序拥有独立的库library和解释器interpreter，而不用与其他Python程序共享统一个library和interpreter。虚拟环境的好处是避免了不同Python程序间的互相影响（共同使用global library 和interpreter），例如程序A需要某个库的1.0版本，而程序B需要同样这个库的2.0版本，如果程序B执行则A就不能执行了。

venv是新创建的虚拟环境的名称。 同时会创建一个与虚拟环境名称相同的文件夹venv, 里面存储了一个独立的Python执行环境。命令virtualenv就可以创建一个独立的Python运行环境，我们还可以加上参数--no-site-packages，这样，已经安装到系统Python环境中的所有第三方包都不会复制过来，这样，我们就得到了一个不带任何第三方包的“干净”的Python运行环境。

(venv)user@machine:~$
